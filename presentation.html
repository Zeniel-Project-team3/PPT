❓ 이런 고민, 있으신가요?

상담 기록과 데이터를 어떻게 하면 제대로 활용할 수 있을까요? — 먼저 현장의 목소리에 공감합니다.

⚠️ 상담 데이터가 제각각이라 활용이 어렵다

"상담사 10명이면 10가지 결과물" — 기록 방식이 달라 소중한 상담 기록이 단순 텍스트 뭉치로만 남고, 데이터로 쌓이지 않는 현실

⚠️ 상담사에 따라 내담자 결과가 달라진다

숙련 상담사와 신입의 노하우 차이로, 같은 내담자도 만나는 사람에 따라 정보의 질과 성공률이 달라지는 불균형

⚠️ 서류·검토에 상담보다 시간이 더 든다

상담 시간보다 긴 서류 작성과 사업 연계 검토 — 상담 본업에 집중하기 어려운 구조

💡 그렇다면, 해결책은?

데이터를 자산으로 바꾸고, 상담 품질을 한 단계 올리는 방법 — 우리의 강점을 구체적으로 제시합니다.

✓ 데이터의 자산화

자유 형식의 상담 일지를 AI가 실시간 분석해 정형 데이터로 변환합니다. 쌓여만 있던 기록을 검색·분석 가능한 자산으로 전환합니다.

✓ 지식의 평준화

신입 상담사도 유사 사례 기반 가이드를 바로 활용할 수 있도록, 베테랑의 노하우를 표준화된 형태로 제공합니다.

⚙️ 어떻게 가능하게 할까요?

세 가지 핵심 기능으로 ‘기록 → 분석 → 추천’을 한 번에

안전하고 스마트한 식별

이름·전화번호만으로 과거 이력과 유사 사례를 안전하게 매칭합니다.

AI 기반 상담 네비게이션

나이·성별·학력·역량을 종합 분석해 최적의 직무·시나리오를 자동 추천합니다.

상담 내역 정형화

자유 형식 일지를 AI가 실시간 분석해 정형 데이터로 변환하고, 자동 요약·DB 저장·직무 추천까지 연계합니다.

📋 실제로는 어떻게 쓰이나요?

상담사의 하루 — 데이터 흐름을 따라가 보세요

1

상담 전 조회

2

3

4

검수 및 마감

생성 데이터 확인 → 원클릭 수정·마감

시스템 아키텍처

데이터 기반의 신뢰 — 검증된 기술 스택으로 안정성을 보장합니다

핵심 기술

PostgreSQL(pgvector)을 활용하여 텍스트 데이터의 의미적 유사도를 초고속으로 검색하여 유사 사례를 매칭 · Spring Boot와 Python(FastAPI)이 Docker 환경에서 협업

보안

Spring Server가 개인정보를 식별화(clientId)하여 AI 서버(openAI)로 전달함으로써 보안 통신 규격 준수

백엔드 시스템 아키텍처

Spring Boot 기반 3계층 아키텍처 · REST API · JPA · PostgreSQL(pgvector) · Spring AI

데이터 흐름

요청 → Controller → Service → Repository/Utility/AI 연동 → PostgreSQL. Spring Boot가 비즈니스 로직·보안을 관리하고, pgvector로 유사도 검색을 수행합니다.

기술 스택

Spring Boot, REST API, JPA, PostgreSQL(pgvector), Spring AI(ChatClient, Embedding), OpenAI, Apache Tika

🏗 4. 시스템 아키텍처 · AI 서비스 추천 API

핵심 메시지: "빠르고, 안전하며, 확장이 용이한 최신 기술의 집약체."

설명: Java Backend가 추천 API를 호출하면, PostgreSQL(pgvector)에서 클라이언트 프로필·임베딩·유사 사례를 조회하고, OpenAI로 임베딩·GPT 추천을 수행합니다. PII 마스킹 후 규칙 기반 + LLM 추천을 병합해 최종 응답을 반환합니다.

세일즈 포인트: pgvector로 유사 사례를 초고속 검색하고, GPT와 규칙 기반을 결합해 빈 결과 시에도 안정적인 추천이 가능합니다.

📊 1. Top-K(유사사례) 추출 결과

핵심 메시지: "막연한 추측이 아닌, 실제 성공 데이터에 기반한 증거를 제시합니다."

설명: 이 화면은 AI가 현재 내담자와 가장 유사한 특징을 가진 과거 취업 성공 케이스를 실시간으로 찾아낸 결과입니다.

AI 추천 취업한 케이스만 추출해서 내담자에게 유리한 상담 자료 제공

나이 성별 학력 직업 훈련 취업 정보 희망 직종
1 28 남 대졸 웹개발 정규직 취업 소프트웨어
2 35 여 고졸 디자인 계약직 UI/UX
3 42 남 대졸 데이터분석 재취업 데이터엔지니어

이러한 데이터를 추출했습니다.

세일즈 포인트: 상담사는 이 데이터를 통해 내담자에게 '당신과 비슷한 역량을 가진 분들이 실제로 이러한 경로로 취업에 성공했습니다'라는 강력한 확신을 줄 수 있습니다.

📝 2. 프롬프트 (AI_프롬프트)

핵심 메시지: "상담사의 전문성을 AI의 논리적인 가이드로 변환합니다."

system_prompt = (
    "너는 고용 상담 AI다. 반드시 JSON만 반환한다. "
    "PII(이름/주민번호)는 절대 생성하거나 복원하지 않는다. "
    "추천 직무는 반드시 두 종류를 모두 채운다: "
    "(1) recommendedJobsByProfile: 나이·성별·학력·역량·전공만 보고(희망직종은 보지 않음) 적합한 직무 3개, 1위~3위. 꼭 3개. "
    "(2) recommendedJobsByDesiredJob: 내담자 희망직종을 반영한 직무 3개, 1위~3위. 꼭 3개. "
    "recommendedTrainings는 위 추천 직무와 연관된 과정명 3개. "
    "expectedSalaryRange, suggestedServices 3개, coreQuestions 3개, recommendedCompanies, reason 한 문장."
)
user_prompt = {
    "mode": mode,
    "input": {
        "age": masked_input.get("age"),
        "gender": masked_input.get("gender"),
        "education": masked_input.get("education"),
        "desiredJobs": desired_jobs_list,
        "competency": masked_input.get("competency"),
        "major": masked_input.get("major"),
    },
    "similarCases": compressed_cases,
    "baseRecommendation": base,
    "outputFormat": {
        "recommendedJobsByProfile": "배열 3개, 나이·성별·학력·역량·전공만 고려한 직무(희망직종 미반영)",
        "recommendedJobsByDesiredJob": "배열 3개, 희망직종 반영한 직무",
        "recommendedTrainings": "배열 3개, 추천 직무와 연관된 과정명",
        "recommendedCompanies": "배열",
        "expectedSalaryRange": "추천 직무 기준 연봉 범위 문자열",
        "suggestedServices": "배열 3개, 중요도순",
        "coreQuestions": "배열 3개, 중요도순",
        "reason": "GPT 판단 근거",
    },
}

설명: AI가 상담 시나리오를 작성할 때 기준이 되는 '지시서'입니다. 내담자 정보는 clientId로 대체 전달하며, 유사 사례와 희망 사항을 결합해 상담 스크립트를 생성합니다.

세일즈 포인트: 단순 질의응답을 넘어 상담 품질 표준화를 위한 최적화된 로직으로 설계되었습니다.

🎯 3. 추천 결과 (직무/급여 제안)

핵심 메시지: "정성적 판단을 넘어선 데이터 기반의 정량적 컨설팅."

설명: AI가 분석한 최종 결과물입니다. 내담자의 학력, 역량, 나이 등을 종합하여 가장 취업 확률이 높은 맞춤형 직무와 예상 급여 수준을 제안합니다.

세일즈 포인트: 희망 직종을 반영하면서도 현실적인 시장 데이터를 매칭함으로써, 내담자의 눈높이를 조절하고 취업 성공률을 비약적으로 높이는 고도화된 컨설팅이 가능해집니다.
